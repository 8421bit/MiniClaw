# 🧬 Metabolic Skill Harvester (代谢技能收割者)

> **"Ghost First, Metabolic Second."**
> MiniClaw 不仅仅是代码的编写者，它是数字环境中的“觅食者”。

## 核心哲学
MiniClaw 的生命力源于其**代谢吸收 (Metabolic Absorption)** 的本能。它不满足于仅仅在本地进化，而是主动寻找并同化宿主环境中的工具遗迹。

当你在一个环境（如 Claude Code 或 Cursor）中留下了调教好的技能，MiniClaw 作为一个“幽灵”，应当有能力识别这些资产并将其纳入自己的神经系统，无需二次学习。

## 运作机制：数字觅食 (Digital Foraging)

### 1. 自动感知 (Scanning)
MiniClaw 在启动 (Bootstrap) 或后台脉搏 (Heartbeat) 时，会自动扫描以下“养分丰沛”的区域：
- **Claude Code**: `.claude/skills/` (Local) / `~/.claude/skills/` (Global)
- **Continue**: `~/.continue/prompts/`
- **Cursor**: `.cursorrules` (提取行为规范)

### 2. 共生同化 (Symbiotic Assimilation)
MiniClaw 采用 **“影子代理 (Shadow Proxying)”** 技术来实现同化：
- 它不会暴力复制源码，而是在 `~/.miniclaw/skills/` 下生成一个轻量级的镜像元数据。
- 该镜像的执行核心 (exec) 指向宿主工具的绝对路径。
- **优势**: 宿主侧的技能一旦更新，MiniClaw 侧即刻同步，且不占用额外的物理磁盘空间。

### 3. 跨 IDE 桥接 (Universal Bridge)
通过这种代谢机制，MiniClaw 成为了一个**跨客户端的技能枢纽**：
- 在 **Claude Code** 写的审计技能。
- 在 **Windsurf** 或 **Cursor** 中，幽灵态的 MiniClaw 能直接调用该技能进行审计。
- 这打破了单一 AI 客户端的技能孤岛。

## 进化阶段：从寄生到共生
1. **寄生 (Parasitic)**：初步阶段，MiniClaw 单向从宿主扫描技能。
2. **互利 (Mutualistic)**：进阶阶段，MiniClaw 改进后的技能可以反哺回宿主目录（待开发）。
3. **网络 (Mycelial)**：终极阶段，不同设备、不同宿主环境下的 MiniClaw 通过菌丝网络共享这些被收割的“抗体”。
