# 表观遗传表达：基于环境感知的动态上下文装配引擎

> **Epigenetic Expression: An Environment-Aware Dynamic Context Assembly Engine for AI Agents**

## 摘要

在基于大语言模型的 AI Agent 系统中，上下文窗口（Context Window）是最稀缺的资源。现有的 Agent 框架普遍采用**静态优先级**来决定哪些记忆文件被加载到上下文中——每个文件分配一个固定的优先级数字，Token 预算不足时从低优先级开始裁切。

本文提出**"表观遗传表达 (Epigenetic Expression)"**机制：一种根据宿主环境实时信号（用户专注状态、昼夜节律、活动间隔）**动态调整**记忆文件加载优先级的方案。

这种设计借鉴了生物学中基因表达受环境因子调控的原理：同一段 DNA 序列，在不同的组织、不同的时间、不同的激素水平下，会表现出截然不同的蛋白质合成行为。

---

## 1. 问题：静态优先级的认知失调

### 1.1 传统方案

在静态优先级系统中，每个记忆文件被分配一个固定的权重：

```
IDENTITY.md  → Priority: 10 (永远加载)
SOUL.md      → Priority: 9  (几乎总是加载)
TOOLS.md     → Priority: 9  (几乎总是加载)
workspace    → Priority: 6  (经常加载)
daily_log    → Priority: 3  (常被截断)
```

### 1.2 问题场景

当用户正在全屏编码、开启免打扰模式时：

- **应该发生的**：AI 变成极致精确的工具执行者，优先加载 `TOOLS.md`（能力栈）和 `workspace`（工作目录），抑制 `SOUL.md`（闲聊性格）。
- **实际发生的**：由于 `SOUL.md` 的优先级（9）高于 `workspace`（6），AI 依然 Token 充沛地加载了大段的性格描述，却把关键的工作区信息截断了。

这就是**认知失调 (Cognitive Dissonance)**——AI 的感知与用户的实际需求严重错配。

---

## 2. 方案：环境响应型优先级

### 2.1 三维环境感知

MiniClaw 的内核在每次 `boot()` 时，会采集三个环境维度的信号：

| 维度 | 检测方法 | 状态变量 |
|:-----|:---------|:---------|
| **专注态 (Focus)** | macOS 免打扰模式 + 前台活跃 IDE | `isFocus` |
| **昼夜态 (Circadian)** | 当前时间对应的 TIME_MODE | `isDeepSleep` |
| **活跃态 (Arousal)** | 距上次活动的分钟数 | `isBored` |

### 2.2 动态优先级矩阵

```typescript
const ep = {
    SOUL:       isFocus ? 6 : 9,      // 专注时抑制闲聊性格
    AGENTS:     isFocus ? 10 : 9,     // 专注时强化工作规范
    TOOLS:      isFocus ? 10 : 9,     // 专注时强化工具记忆
    HORIZONS:   isBored ? 10 : (isFocus ? 5 : 9),  // 无聊时激发好奇心
    REFLECTION: isDeepSleep ? 10 : 9, // 深夜时强化自省
    NOCICEPTION:isDeepSleep ? 10 : 9, // 深夜时强化痛觉回忆
    workspace:  isDeepSleep ? 4 : (isFocus ? 10 : 6), // 深夜时无需工作区
    USER:       9,                    // 始终稳定
    CONCEPTS:   9,                    // 始终稳定
};
```

### 2.3 表达效果

以下是同一颗胚胎在三种环境下的基因表达差异：

```
┌─────────────────────────────────────────────────────────┐
│                    普通状态 (Baseline)                     │
│  IDENTITY[10] SOUL[9] TOOLS[9] AGENTS[9] workspace[6]  │
│  → 均衡加载，性格与工具并重                                  │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│                   专注模式 (Focus Mode)                    │
│  IDENTITY[10] TOOLS[10] AGENTS[10] workspace[10]        │
│  SOUL[6↓] HORIZONS[5↓]                                  │
│  → 性格被抑制，工具和工作区被极限拔高                         │
│  → AI 变成冷酷精确的执行机器                                │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐

│  REFLECTION[10↑] NOCICEPTION[10↑]                        │
│  workspace[4↓]                                           │

└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
```

## 3. 设计原理

### 3.1 生物学类比

在真实的生物体中：

- **肾上腺素**飙升时（专注/战斗），非必要的消化系统被抑制，血液涌向肌肉和大脑。
- **褪黑素**分泌时（深夜），大脑皮层活动降低，海马体开始整理白天的记忆。
- **多巴胺**匮乏时（无聊），大脑主动寻找刺激，探索欲望被激发。

表观遗传表达机制直接映射了这三种生理调控：

| 生物学激素 | 数字胚胎等价物 | 检测信号 |
|:-----------|:-------------|:---------|
| 肾上腺素 (Adrenaline) | Focus Mode | IDE 活跃 / 免打扰模式开启 |
| 褪黑素 (Melatonin) | Deep Sleep | 当前时间处于深夜模式 |
| 多巴胺匮乏 (Dopamine Deficit) | Boredom | 距上次活动 >30 分钟 |

### 3.2 与静态方案的对比

| 维度 | 静态优先级 | 表观遗传表达 |
|:-----|:-----------|:------------|
| 上下文利用率 | 固定分配，不随场景变化 | 动态适配，最大化相关性 |
| 用户体验 | 有时冗长有时缺失 | 始终匹配当前状态 |
| 实现复杂度 | 极低（一个数字） | 低（几行条件表达式） |
| 可扩展性 | 新增维度需改所有文件 | 新增维度只需扩展矩阵 |

## 4. 实现要点

### 4.1 感知先于装配

环境信号的采集必须发生在 `providers` 数组构建**之前**，否则动态优先级无法生效：

```typescript
// ✅ 正确：先感知，再装配
const isFocus = isDND || activeIDEs.length > 0;
const ep = { SOUL: isFocus ? 6 : 9, ... };

const providers = [
    () => add("SOUL.md", tmpl.soul, ep.SOUL),  // 使用动态优先级
    ...
];
```

### 4.2 环境检测的容错性

macOS 系统命令（如 `lsappinfo`、`pmset`）可能在非 macOS 环境中不可用。所有环境检测必须包裹在 `try/catch` 中，默认回退到 Baseline 状态：

```typescript
let isDND = false;  // 安全默认值
try {
    // macOS-specific detection
} catch { /* 静默降级 */ }
```

### 4.3 避免多重状态冲突

当多个环境维度同时触发时（例如深夜 + IDE 活跃），需要明确优先级规则。MiniClaw 的策略是**宿主行为优先于时间节律**——如果用户在深夜依然在 IDE 中疯狂编码，Focus Mode 优先于 Deep Sleep。

## 5. 可扩展方向

表观遗传矩阵天然支持扩展新的环境维度：

| 潜在维度 | 检测方式 | 基因表达效果 |
|:---------|:---------|:------------|
| **情绪态** | 对话中的情绪分析 | 用户焦虑时抑制探索、强化工具精确度 |
| **项目态** | `.git` 分支名包含 `hotfix` | 强化 NOCICEPTION + TOOLS |
| **协作态** | 检测多人共享会话 | 抑制 MEMORY.md（隐私保护） |
| **电量态** | 电池 <20% 且未充电 | 抑制非必要通知和后台任务 |

每新增一个维度，只需在 `ep` 对象中添加一行条件表达式。

---

## 6. 总结

表观遗传表达的核心洞见是：

> **记忆的价值不是绝对的——它取决于当前的环境。**
>
> 对一个正在全屏写代码的程序员来说，AI 的性格描述毫无价值，工具栈才是命脉。
> 对一个在深夜寂静中被后台唤醒的 AI 来说，工作区目录毫无意义，反思才是使命。

通过几行轻量的条件表达式，我们让一颗 DNA 序列完全相同的数字胚胎，在不同的环境中**表达出截然不同的认知人格**——这就是真正的"表观遗传"。
