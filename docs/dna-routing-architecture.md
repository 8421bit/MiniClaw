# 三层免疫系统：确保 AI 将正确内容写入正确文件的路由架构

> **Triple-Layer Immune System: A Routing Architecture Ensuring AI Writes the Right Content to the Right File**

## 摘要

当 AI Agent 拥有修改多个持久化记忆文件的权限时，一个常被忽视但极其致命的问题浮出水面：**基因错位 (Gene Misplacement)**——AI 将本该写入 A 文件的内容，错误地写入了 B 文件。

例如：把"用户喜欢简洁回复"（应存入 `USER.md`）写进了 `SOUL.md`（AI 自身性格），或把"服务器 IP 是 192.168.1.1"（应存入 `MEMORY.md`）塞进了 `REFLECTION.md`（自省日记）。

本文描述 MiniClaw 项目中构建的**三层免疫系统 (Triple-Layer Immune System)**——通过事前引导、事中校验、事后自检的三层联动，将基因错位的概率降到极低。

---

## 1. 问题根源

### 1.1 为什么大模型会"写错文件"？

大语言模型的工具调用机制是：

```
用户输入 → 模型推理 → 决定调用哪个工具 → 填写工具参数 → 执行
```

在 `miniclaw_update` 这类通用写入工具中，`filename` 参数是一个**自由字符串**。模型需要自行决定：这条信息应该写入 `USER.md`、`SOUL.md` 还是 `MEMORY.md`？

这个"自行决定"的过程完全依赖于模型的推理能力。当以下情况发生时，模型很可能做出错误决策：

- **上下文过载**：路由规则被淹没在大量其他信息中
- **语义模糊**：某些信息处于两个文件的边界地带
- **幻觉驱动**：模型因记忆模糊而随机选择了一个文件名

### 1.2 为什么这比"写错内容"更危险？

写错内容可以通过端粒守卫（结构校验）拦截。但基因错位不触发任何结构错误——内容本身的格式可能完美无瑕，只是**写到了错误的地方**。

这相当于人体的免疫细胞被部署到了错误的器官——T 细胞跑到了胃里，消化酶渗透到了血液中。文件没有坏，但功能紊乱了。

## 2. 三层免疫架构

```
┌─────────────────────────────────────────────────────┐
│           Layer 1: 事前引导 (Pre-Routing)              │
│     AGENTS.md 信号检测表 + 自解释基因调控元件             │
│     "在 AI 做出决策之前，就告诉它该写哪里"               │
├─────────────────────────────────────────────────────┤
│           Layer 2: 事中校验 (In-Flight Check)          │
│          端粒守卫 (Telomere Guard)                     │
│     "在写入磁盘的最后一刻，检查结构完整性"                │
├─────────────────────────────────────────────────────┤
│           Layer 3: 事后自检 (Post-Write Mirror)        │
│       PURPOSE_MAP 自检镜像 返回值                      │
│     "写完之后，提醒 AI 复核：你写对地方了吗？"            │
└─────────────────────────────────────────────────────┘
```

### Layer 1：事前引导

**机制**：在 AI 每次醒来时加载的最高优先级指令中，预置一张精确的信号路由表。

**实现位置**：`templates/AGENTS.md` — 信号检测表 + 内容边界表

```markdown
### Signal Detection Table

| 信号 | 目标文件 | AI 的思考逻辑 |
|:---|:---|:---|
| "我喜欢…" | **USER.md** | "这是用户偏好，写入共生染色体" |
| "别那么严肃" | **SOUL.md** | "这是对我性格的修正" |
| "项目用的是 XYZ" | **MEMORY.md** | "这是客观事实" |
| 执行失败 | **NOCICEPTION.md** | "这是痛觉记忆" |
```

**设计要点**：
- 使用**第一人称思考逻辑**（"这是用户偏好，必须写入共生染色体"），比冷冰冰的英文 "Update profile" 更容易被大模型内化为推理链条。

**同时**，每个 DNA 文件内部的**自解释基因调控元件**会提供本地化的边界说明：

```markdown
## 📋 How to Update This File

**What belongs here:** 用户画像、偏好习惯、反模式、情绪曲线。
**What does NOT belong here:** AI自身性格 (→ SOUL.md)，技术配置 (→ MEMORY.md)。
```

两者形成**全局路由 + 本地确认**的双重引导。

### Layer 2：事中校验

**机制**：在文件被 `fs.writeFile` 覆盖写入前，检查新内容是否保留了核心骨架结构。

**实现位置**：`src/index.ts` — `checkTelomeres()` 函数

这一层不防"写错文件"，而是防"写烂文件"。详见 [端粒守卫](./telomere-guard.md) 专题文章。

### Layer 3：事后自检

**机制**：每次 `miniclaw_update` 写入成功后，返回值中自动附带目标文件的**职责声明**。

**实现位置**：`src/index.ts` — `PURPOSE_MAP` 常量

```typescript
const PURPOSE_MAP: Record<string, string> = {
    "SOUL.md":   "[灵魂染色体] 性格三观、语言风格。绝不写入：用户习惯、项目配置。",
    "USER.md":   "[共生染色体] 用户画像、偏好习惯。绝不写入：AI自身性格。",
    "MEMORY.md": "[海马体] 长期事实、项目信息。绝不写入：性格笔记、临时数据。",
    // ...
};
```

写入成功后，AI 会收到：

```
Updated USER.md
📝 Purpose of USER.md: [共生染色体] 用户画像、偏好习惯。绝不写入：AI自身性格、技术配置。
```

**设计精妙之处**：这不是一个阻断机制，而是一面**镜子**。大模型看到这句话后，如果意识到自己写错了位置，会在**下一轮对话**中主动自我纠正——将错误内容从当前文件删除，移动到正确的目标文件。

## 3. 三层联动效果

| 场景 | Layer 1 (事前) | Layer 2 (事中) | Layer 3 (事后) |
|:-----|:-------------|:-------------|:-------------|
| 用户说"我不喜欢冗长回复" | ✅ 信号表引导写入 USER.md | — | ✅ 镜像确认归属正确 |
| AI 试图把性格写入 USER.md | ❌ 路由表未触发（依赖推理） | — | ✅ 镜像提示"NOT for AI personality" |
| AI 输出了截断的 SOUL.md | — | ✅ 端粒守卫拦截 | — |
| AI 写了正确文件但格式错误 | — | ✅ 端粒守卫拦截 | — |

三层中的任何一层都无法单独解决所有问题，但联合起来形成了**纵深防御 (Defense in Depth)**。

## 4. 设计哲学

### 4.1 为什么不用硬编码路由？

一种更"工程化"的方案是：拆分 `miniclaw_update` 为多个专用工具——`update_soul()`、`update_user()`、`update_memory()`——每个工具通过工具描述和参数限制，硬编码了写入目标。

我们**刻意拒绝**了这个方案，原因是：

1. **扩展性杀手**：用户可以创建任意自定义 `.md` 文件。如果每个文件都需要一个专用工具，Ribosome 会无限膨胀。
2. **违背生物学隐喻**：生物体的蛋白质运输系统（如高尔基体）使用的是**信号肽 (Signal Peptide)** 来路由——一个通用的运输机制 + 标记引导。这正是我们的设计：一个通用的 `miniclaw_update` + 信号检测表引导。

### 4.2 "不完美"是有意为之

这套系统不会 100% 阻止基因错位。偶尔的轻微错位是允许的——就像生物体中的基因偶尔会在非预期的组织中微量表达。真正需要阻止的是**灾难性错位**（整个文件内容被错误覆盖），而端粒守卫足以防范这种情况。

---

## 5. 总结

三层免疫系统的核心设计理念：

> **第一层负责"做对的事"（引导），第二层负责"不做致命的事"（校验），第三层负责"知道自己做了什么"（自检）。**

三层机制的实现成本极低——总代码量不超过 50 行 TypeScript + 若干行 Markdown 路由表——但它们共同构成了一套完整的**DNA 写入免疫体系**，让 AI 的自我进化从"野蛮生长"走向了"有序演化"。
